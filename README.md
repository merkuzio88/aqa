# Тестовое задание: QA Automation (API)

Автоматизированное тестирование веб-сервиса с использованием мок-сервера для имитации внешних зависимостей.

📊 **Allure Отчет:** [Ссылка на отчет](https://merkuzio88.github.io/aqa/)

## 🛠 Стек технологий
* **Java 17**
* **JUnit 5** — тестовый фреймворк
* **RestAssured** — HTTP-клиент для API тестов
* **WireMock** — мокирование внешних сервисов (`/auth`, `/doAction`)
* **Allure Report** — генерация понятных отчетов для нетехнических специалистов
* **Maven** — сборка проекта

## 🚀 Инструкция по запуску

### 1. Запуск тестируемого приложения (Docker)
Для изоляции и корректной работы сети рекомендуется запускать приложение в Docker.
В терминале выполните команду (убедитесь, что `internal-0.0.1-SNAPSHOT.jar` находится в нужной папке):

```bash
docker run --rm -it -p 8080:8080 -v ${PWD}:/app -w /app eclipse-temurin:17-jre java -jar -Dsecret=qazWSXedc -Dmock=http://host.docker.internal:8888/ internal-0.0.1-SNAPSHOT.jar
```
*(Для Linux замените `host.docker.internal` на `localhost` и добавьте `--network host`)*

### 2. Запуск тестов
В корне проекта выполните команду:
```bash
mvn clean test
```

### 3. Просмотр отчета Allure
После успешного (или неуспешного) прогона тестов сгенерируйте отчет:
```bash
mvn allure:serve
```
Отчет автоматически откроется в браузере. В нем описаны подробные шаги (Steps) на русском языке, а также прикреплены логи запросов и ответов.

## 🐞 Заметки по тестированию (Найденный дефект)

В ходе автоматизации было выявлено несоответствие поведения приложения и требований документации.

**Описание:**
В ТЗ указано, что `token` — это строка длиной 32 символа, состоящая из `A-Z0-9`.
Однако, при отправке токена, содержащего буквенные символы (например, `USER1234567890123456789012345678`), приложение возвращает ошибку `400 Bad Request`. Приложение успешно принимает только токены, состоящие исключительно из цифр `0-9`.

**Workaround в тестах:**
Чтобы избежать падения тестов по не связанным с логикой причинам, генератор случайных токенов в `BaseTest.java` временно настроен на генерацию валидных токенов только из цифр (0-9).
